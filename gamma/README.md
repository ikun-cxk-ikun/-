# 技术思维处理器 - Web前端

一个基于Flask的技术面试思维分析工具，能够实时分析技术问题并提供AI驱动的思考洞察。

## 📚 功能特性

### 🔥 核心功能
- **实时思维分析**: 边输入边分析，即时提取技术关键词和概念
- **智能技术过滤**: 自动识别和过滤非技术相关问题
- **多模型支持**: 支持Claude、GPT-4等多种大语言模型
- **上下文记忆**: 保持对话上下文，提供连贯的分析体验
- **可视化界面**: 现代化Web界面，实时显示思考过程

### 📊 界面功能
- **主页面**: 实时输入和思考展示
- **配置页面**: API密钥和模型参数设置
- **历史记录**: 查看、搜索和导出处理历史
- **状态监控**: 实时显示处理器运行状态

## 🚀 快速开始

### 1. 环境要求
```bash
Python 3.7+
Flask
openai
requests
```

### 2. 安装依赖
```bash
pip install flask openai requests
```

### 3. 获取API密钥
- 访问 [OpenRouter](https://openrouter.ai/) 获取API密钥
- 支持调用Claude、GPT-4等多种模型

### 4. 启动应用
```bash
python app.py
```

### 5. 访问界面
打开浏览器访问: `http://localhost:5000`

## 💡 使用指南

### 第一次使用
1. 访问 **配置页面** 输入API密钥
2. 选择合适的AI模型（推荐Claude Sonnet 4）
3. 点击"保存并启动"初始化处理器
4. 返回主页开始分析技术问题

### 实时分析体验
1. 在主页的文本框中输入技术问题
2. 系统会实时分析并显示：
   - 识别的技术关键词
   - 相关技术概念
   - 思考摘要
   - 可能的后续问题

### 历史记录管理
- 查看所有处理过的问题
- 按状态、日期过滤记录
- 搜索特定问题或关键词
- 导出历史数据为CSV格式

## 🛠️ 高级配置

### API调用优化
- **调用间隔**: 控制API请求频率（默认1.5秒）
- **上下文窗口**: 设置历史记录数量（默认5条）
- **实时更新**: 调整部分思考触发间隔

### 技术过滤设置
- 启用/禁用技术问题过滤
- 自动识别非技术相关内容
- 可自定义技术关键词库

## 🔧 技术架构

### 后端架构
```
Flask Web应用
├── app.py              # 主程序和路由
├── processor_module/   # 核心处理模块
│   ├── thought_processor.py  # 思维处理器
│   └── __init__.py
├── templates/          # HTML模板
│   ├── base.html
│   ├── index.html
│   ├── config.html
│   └── history.html
└── static/            # 静态资源
```

### 核心组件
- **APITransformer**: AI模型接口封装
- **TechnicalFilter**: 技术问题过滤器
- **TechnicalThoughtProcessor**: 核心思维处理器

## 📱 使用技巧

### 最佳实践
1. **逐步输入**: 像平时说话一样逐字输入，体验实时分析
2. **技术用词**: 使用准确的技术术语获得更好的分析结果
3. **上下文**: 在同一会话中提问相关问题，利用上下文记忆
4. **模型选择**: 根据需求选择合适的AI模型

### 问题示例
```
• 什么是分布式系统的CAP理论？
• 如何优化MySQL数据库的查询性能？
• 解释一下TCP和UDP协议的区别？
• Python中装饰器的工作原理是什么？
• 什么是Docker容器化技术？
```

## 🐛 常见问题

### API相关
- **连接失败**: 检查网络连接和API密钥
- **调用限制**: 调整API调用间隔避免频率限制
- **模型错误**: 确认选择的模型在API密钥权限内

### 功能相关
- **无法识别**: 确认问题包含技术关键词
- **分析缓慢**: 调整处理器参数或选择更快的模型
- **历史丢失**: 数据保存在内存中，重启会清空
