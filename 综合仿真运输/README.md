# 综合运输车辆仿真模拟系统

基于高德地图API开发的综合运输车辆仿真模拟系统，用于模拟各类运输车辆在地图上的行驶轨迹和状态。

## 功能特点

- 🗺️ 基于高德地图API实现地图可视化
- 🚚 支持多种车辆类型（货车、公交车、小汽车）
- 📊 实时显示车辆位置、速度、方向等状态
- 🎯 车辆路径规划和仿真运动
- 🔄 WebSocket实时数据传输
- 🎨 现代化UI界面设计
- 📱 响应式布局，支持不同设备

## 技术栈

- **后端**: Node.js, Express.js
- **实时通信**: Socket.IO
- **地图服务**: 高德地图JavaScript API
- **前端**: HTML5, CSS3, JavaScript
- **环境管理**: dotenv

## 快速开始

### 前置要求

- Node.js 14.x 或更高版本
- npm 或 yarn
- 高德地图API密钥（可选，系统提供模拟数据）

### 安装步骤

1. 克隆项目（如果适用）

2. 安装依赖
   ```bash
   npm install
   ```

3. 配置环境变量
   - 复制 `.env` 文件中的 `AMAP_KEY` 替换为您的高德地图API密钥
   - 或者直接使用模拟数据（无需修改）

4. 启动服务器
   ```bash
   npm start
   ```
   或开发模式：
   ```bash
   npm run dev
   ```

5. 访问系统
   打开浏览器，访问 http://localhost:3000

## 主要模块说明

### 1. 车辆仿真核心（VehicleSimulation）
- 管理多辆车辆的仿真过程
- 根据配置生成随机路线或使用高德地图API规划路线
- 定期更新车辆位置并广播给客户端

### 2. 车辆模型（Vehicle）
- 定义车辆属性（ID、类型、位置、速度、方向等）
- 实现车辆运动逻辑（基于路线或随机运动）
- 计算距离、方向和位置更新

### 3. 高德地图服务（AMapService）
- 封装高德地图API调用
- 提供路线规划、地理编码、交通状况查询等功能
- 当API密钥未配置时提供模拟数据

### 4. 前端可视化
- 地图显示和交互
- 车辆标记和轨迹展示
- 实时数据统计和车辆列表
- 仿真控制（开始、停止、重置）

## 配置说明

在 `.env` 文件中可以配置以下参数：

- `AMAP_KEY`: 高德地图API密钥
- `PORT`: 服务器端口（默认3000）
- `HOST`: 服务器主机（默认localhost）
- `SIMULATION_INTERVAL`: 仿真更新间隔（毫秒，默认1000）
- `VEHICLE_COUNT`: 初始车辆数量（默认10）

## 注意事项

1. **高德地图API密钥**：
   - 系统可以在没有API密钥的情况下运行（使用模拟数据）
   - 如需使用真实地图数据，请注册高德地图开发者并申请API密钥

2. **浏览器兼容性**：
   - 推荐使用 Chrome、Firefox、Safari、Edge 等现代浏览器

3. **性能优化**：
   - 大量车辆可能导致性能下降，建议根据设备性能调整车辆数量
   - 轨迹历史记录限制为100个点，以减少内存占用

## 开发指南

### 添加新功能

1. 在 `src/models/` 中添加新的数据模型
2. 在 `src/simulation/` 中扩展仿真逻辑
3. 在 `src/api/` 中添加新的API服务
4. 修改 `public/` 中的前端文件以添加新的UI元素和交互

### 调试

- 使用 `npm run dev` 启动开发服务器，支持热重载
- 前端控制台可以查看Socket.IO连接状态和数据
- 后端控制台输出仿真状态和错误信息
